Chapter 1 Introduction
1.1 Project Title
Smart AI-Based Structural Bug Detection and Prevention System Using Machine Learning
and LangGraph
1.2 Introduction
An AI-based system design that reviews the source code and the structure of the software
projects and warns developers about the potential structural problems and suggests certain
measures to prevent them.
This system is different from most Debugging Tools that only catch syntax errors or crashes
after the code runs. This aspects of the system examines how the code is organized: how the
modules are dependent on each other, whether there are circular dependencies, coupling
and cohesion [1][2], or components that are likely become hard to maintain.
When the system scans a GitHub repository using GitHub OAuth, it runs static analysis and
builds a dependency map[3] of the project. It then extracts useful metrics about
architectural and design. A machine learning model then predict which part of the system
are risky and likely to cause bugs in the future.
In addition, this system also offers multiple subscription tiers such as Free tier and Pro tier,
where the advanced feature will be unlocked through paid plans which is support by online
payment integration using razorpay[9].
1.3 Problem Statement
Most the existing debugging tools are largely reactive which means they detect problems
only after failure occur during testing or deployment phases. Manual architectural reviews
are slow or time consuming, costly, and prone to human oversight.
At present there is no comprehensive automated platform that:
 Continuously analyses a software architecture.
 Predicts future structural problems
2
 Identifying architectural anti patterns.
 Provide clear and explainable, AI-driven recommendations.
 Visualizes of how dependencies evolve over time.
Hence a predictive and intelligent system is required to proactively monitor software design and help
maintain quality of the software.
1.4 Objective
The main objective of this project is to design and develop an AI-driven based system that
can predict the architectural risk, automatically analyse the structure of the software project
and provide explainable recommendations to prevent future defects.
Objectives:
1. By performing statistical analysis on complete software repositories, in order to
understand module structure, dependencies and design of the system.
2. Extracting architectural and complexity metrics such as coupling and cohesion.
3. To make a bug predicting model based on machine learning in order to avoid system
failure.
4. To detect anti-structural patterns like circular dependencies, God files, tight coupling
and broken architectural layers.
5. To design a LangGraph based reasoning system that identifies and detect risk and
suggests actions which are recommended to developers.
6. Visualizing project structure and predict risk levels using interactive graph and
analytical dashboards.
7. To integrate the system with GitHub repositories so that software project can be
scanned automatically and generate reports.
1.5 Scope and Limitation
Scope
1. Static structural analysis.
2. Prediction using Machine-Learning
3. AI driven architectural recommendation.
4. GitHub repository integration.
3
5. Interactive visualization dashboard.
Limitations
1. The system does not analyse program execution or runtime failures.
2. Syntax and compilation error are outside of scope for the current implementation.
3. Machine learning prediction it depends strongly on the quality and diversity of
training data.
4. Language specific parser is needed, which limits immediate multiple programming
language.
5. Automatic code changes are not included in the current implementation.
6. Performance may not be guaranteed for extremely large repositories without
distributed processing.
7. The System cannot remove any defects and it mainly focuses on lowering or reducing
architectural risk.

CHAPTER 3
Proposed System /System Design
3.1 System Overview
The proposed system is a web-based intelligent platform that uses machine learning and
graph-based analysis to assess the structural quality of software projects and forecast
possible architectural risks. Developers can supply a GitHub repository link or upload a
project folder using the system. After receiving the repository, the system uses static
analysis[3] to look at how the source code is organized and find module dependencies.
In order to extract architectural metrics like module size, coupling, cohesion, fan-in, and fan-
out, the analysis engine builds a dependency graph of the project. A machine learning
model analyses these features to forecast the likelihood of future structural flaws in various
system modules [3].
A LangGraph[8]-based reasoning component evaluates the anticipated risk and offers
explanations and preventive suggestions to improve interpretability. The developers receives
the results via an interactive web dashboard that displays dependency graphs, risk levels,
and recommendations. These are developed using React [4], while backend services
responsible for analysis and processing are implemented using FastAPI [5].
MongoDB and Neo4j databases [6][7] are used by the system to store user data, scan results,
and dependency information. This system overall helps developers proactively detect
structural issues in software architecture and take corrective measures before they lead to
major defects.
3.2 Functional Module
The Proposed System is made up of a number of function modules that cooperates to carry
out the risk prediction, recommendation generation, and repository analysis.
Modules
1. User Authentication:
12
User login and registration are handled by this module.
It saves user data in the database and validates user credentials.
Additionally, it controls subscription status and user sessions.
2. Project Scanner:
The User can either Upload the project folder or a link to a GitHub repository.
It finds all of the project's modules, packages, classes, and directories by scanning
every source file.
3. Engine for Static Analysis:
The static analysis engine studies the structure of the source code without executing
it. It identifies relationships between modules and detects architectural problems
such as tight coupling and circular dependencies.
4. Feature Extratcion:
This module transform the raw analysis results into measurable metrics that can be
used by the machine learning model.
5. Risk Prediction for Machine Learning:
This module checks the evaluated extracted features and calculates the likehood of
structural error in each project module using a trained machine learning model.
6. AI Reasoning:
This module interprets the prediction results and give and explanation for the risk
that had been detected.
7. Visualize and Reporting:
This module provides the results to the users through the dashboard and it also
provide graph and the risk be download for the future analyse.
8. Data Storage Module:
This module stores user information, project scans, and analysis results.
MongoDB is used for storing data and Neo4j stores dependency graph of the
software system.

3.3.5 Proposed Algorithm
Steps and Process
1. Start the system.
2. User logs into the systems.
3. User can upload a project or connects to a GitHub repository.
4. The System scans all source code files.
5. The system identifies modules in the project.
6. Static analysis is performed to detect dependencies between modules.
7. Architectural metrics such as coupling and fan-in/fan-out are calculated.
8. These metrics are sent to the machine learning model.
9. The model predicts the risk level in each module.
10.The LangGraph reasoning engine analyzes the prediction results.
11. The system generates explanations and suggest improvement.
12. The results are then stored in the database.
13. The dashboard displays risk analysis.
14. the user can download the report.
15. End the process.
